# Инструкция по составлению артикулов KOMUI

Документ описывает единый шаблон артикулов для товаров KOMUI. Брендовый префикс в артикуле не используется.

## 1. Общий принцип

Артикул должен однозначно описывать конкретную товарную вариацию:

- дизайн;
- носитель;
- тип нанесения;
- цвет;
- версию макета;
- размер.

Для худи дополнительно указываются посадка и ткань, потому что худи могут быть обычные или укороченные, с начесом или без начеса.

## 2. Шаблон для футболок и свитшотов

```text
DESIGN-GARMENT-DECOR-COLOR-VERSION-SIZE
```

Пример:

```text
D016-TSH-PRT-WGRY-V02-M
```

Расшифровка:

```text
D016  - код дизайна
TSH   - футболка
PRT   - принт
WGRY  - вареный серый
V02   - версия дизайна/макета
M     - размер
```

## 3. Шаблон для худи

```text
DESIGN-HDY-DECOR-COLOR-FIT-FABRIC-VERSION-SIZE
```

Пример:

```text
D002-HDY-EMB-BLK-REG-FLC-V01-M
```

Расшифровка:

```text
D002  - код дизайна
HDY   - худи
EMB   - вышивка
BLK   - черный
REG   - обычная посадка
FLC   - с начесом
V01   - версия дизайна/макета
M     - размер
```

## 4. Коды сегментов

### DESIGN

Код дизайна задается один раз и не меняется при переносе дизайна на другой носитель, цвет или размер.

Формат:

```text
D001, D002, D003...
```

Пример:

```text
D016
```

Если один и тот же принт Сатору Годжо используется на белой футболке, вареной серой футболке и худи, код дизайна остается один и тот же. Меняются только остальные сегменты.

### GARMENT

```text
TSH - футболка
HDY - худи
SWT - свитшот
```

### DECOR

```text
PRT - принт
EMB - вышивка
```

### COLOR

```text
BLK  - черный
WHT  - белый
WGRY - вареный серый
WBEG - вареный бежевый
BLU  - синий
```

Если появится новый цвет, добавляй новый короткий код в том же стиле. Например:

```text
GRN - зеленый
RED - красный
NVY - темно-синий
```

### FIT

Используется только для худи.

```text
REG - обычное худи
CRP - укороченное худи
```

### FABRIC

Используется только для худи.

```text
FLC - с начесом
NF  - без начеса
```

### VERSION

Версия нужна, когда дизайн, макет, посадка принта, размер вышивки или важная конструктивная деталь меняется, но товар остается тем же дизайном.

```text
V01 - первая версия
V02 - вторая версия
V03 - третья версия
```

Когда увеличивать версию:

- изменился сам принт или вышивка;
- изменилось расположение принта/вышивки;
- изменился размер нанесения;
- добавлена новая версия макета;
- поменялась важная конструктивная особенность товара внутри той же группы.

Когда НЕ увеличивать версию:

- добавился новый размер;
- добавился новый цвет;
- тот же дизайн перенесен на другой носитель;
- изменилась цена;
- изменилась фотография товара.

### SIZE

```text
XS
S
M
L
XL
2XL
3XL
```

В артикулах лучше использовать `2XL`, а не `XXL`, потому что так короче и удобнее масштабировать размерную сетку.

## 5. Примеры

### Футболка с принтом

```text
D016-TSH-PRT-WGRY-V02-M
```

Смысл: дизайн `D016`, футболка, принт, вареный серый, версия 2, размер M.

### Футболка с вышивкой

```text
D004-TSH-EMB-WHT-V01-L
```

Смысл: дизайн `D004`, футболка, вышивка, белый цвет, версия 1, размер L.

### Свитшот с вышивкой

```text
D002-SWT-EMB-BLK-V01-XL
```

Смысл: дизайн `D002`, свитшот, вышивка, черный цвет, версия 1, размер XL.

### Обычное худи с начесом

```text
D002-HDY-EMB-BLK-REG-FLC-V01-M
```

Смысл: дизайн `D002`, худи, вышивка, черный цвет, обычная посадка, с начесом, версия 1, размер M.

### Обычное худи без начеса

```text
D002-HDY-EMB-BLU-REG-NF-V02-S
```

Смысл: дизайн `D002`, худи, вышивка, синий цвет, обычная посадка, без начеса, версия 2, размер S.

### Укороченное худи с начесом

```text
D018-HDY-EMB-BLK-CRP-FLC-V01-L
```

Смысл: дизайн `D018`, худи, вышивка, черный цвет, укороченная посадка, с начесом, версия 1, размер L.

## 6. Правила для новых товаров

1. Сначала определи дизайн.
   Если это новый принт или новая вышивка, создай новый код `D###`.

2. Определи носитель.
   Футболка = `TSH`, худи = `HDY`, свитшот = `SWT`.

3. Определи тип нанесения.
   Принт = `PRT`, вышивка = `EMB`.

4. Определи цвет.
   Используй существующий код цвета или добавь новый в справочник.

5. Если это худи, добавь посадку и ткань.
   Например: `REG-FLC`, `REG-NF`, `CRP-FLC`, `CRP-NF`.

6. Определи версию.
   Для нового дизайна обычно `V01`. Если это обновленная версия уже существующего дизайна, используй `V02`, `V03` и так далее.

7. Добавь размер.
   Один размер = один итоговый артикул.

## 7. Чеклист перед созданием артикула

- Дизайн уже существует или нужен новый `D###`?
- Носитель указан верно?
- Принт и вышивка не перепутаны?
- Цвет соответствует фактическому товару и фото?
- Для худи указаны посадка и ткань?
- Версия действительно нужна новая или это тот же макет?
- Размер указан в стандартном формате?
- Такой артикул еще не существует?

## 8. Рекомендация по хранению

В таблице товаров лучше хранить:

- старый Ozon `offer_id`;
- новый артикул по шаблону;
- Ozon SKU;
- product_id Ozon;
- размер;
- дизайн-код;
- носитель;
- цвет;
- тип нанесения;
- для худи: посадку и ткань.

Это позволит безопасно перейти со старых Ozon-артикулов на новые, не потеряв связь с историческими заказами и аналитикой.
