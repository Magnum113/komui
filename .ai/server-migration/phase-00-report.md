## Риски

1. UFW выключен, nftables имеет policy accept.
2. SSH разрешает root login и password authentication.
3. За 24 часа зарегистрировано около 2135 неуспешных SSH-попыток.
4. Swap отсутствует.
5. `api.komui.ru` уже является частью действующего production-контура.
6. На сервере есть существующие процессы, которые нельзя останавливать.
7. Основное место на диске занимают OpenClaw и npm cache; удаление не разрешено.
8. Пакеты требуют обновления, но обновление может затронуть действующие сервисы.

## Ограничения следующего этапа

- Не изменять текущий vhost `api.komui.ru`.
- Не перезапускать OpenClaw, Kaiten, Xray и Nginx без отдельного плана.
- Staging размещать на отдельном hostname или локальном порту.
- Перед изменением SSH сначала проверить второй активный административный доступ.
- Не очищать `/home/admin`, `/root/.npm` и другие пользовательские данные.
- Не устанавливать PostgreSQL до фиксации backup и ресурсных лимитов.

- На сервере выполнялись только команды чтения.

- Конфигурация, firewall, SSH, DNS и сервисы не изменялись.
- Production Supabase/Vercel не затронуты.
